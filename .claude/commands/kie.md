kie.ai API 레퍼런스 조회 스킬. 자연어로 질문하면 KIE_API_REFERENCE.md에서 해당 정보를 찾아 답변한다.

**Instructions:**

1. `KIE_API_REFERENCE.md` 파일을 읽는다.

2. 사용자 질문을 분석하여 해당 섹션을 찾는다:
   - "음악 생성" / "곡 만들기" → 섹션 2.1
   - "연장" / "extend" → 섹션 2.2
   - "보컬" / "vocal" → 섹션 2.3
   - "카라오케" / "타임스탬프" → 섹션 2.4
   - "가사" / "lyrics" → 섹션 3
   - "비디오" / "MV" / "뮤직비디오" → 섹션 4
   - "LLM" / "채팅" / "gemini" → 섹션 5
   - "모델" / "model" → 섹션 6
   - "에러" / "오류" → 섹션 7
   - "폴링" / "polling" → 섹션 8
   - "콜백" / "webhook" → 섹션 9
   - "가격" / "크레딧" / "pricing" → 섹션 1
   - "전체" / "목록" / "리스트" → 섹션 10
   - 번호 직접 지정 (예: "3번") → 해당 섹션

3. 찾은 섹션의 내용을 그대로 출력한다:
   - 엔드포인트, 파라미터 테이블, 요청/응답 예시 포함
   - 관련 주의사항이나 팁도 함께 표시

4. 질문이 여러 섹션에 걸치면 관련 섹션 모두 표시한다.

5. 실제 코드에서 사용 중인 패턴도 함께 안내한다:
   - `index.html`에서 해당 API를 호출하는 함수명
   - `api/` 디렉토리의 프록시 파일

**Examples:**
- "가사 생성 API 알려줘" → 섹션 3 전체 출력
- "3번" → 섹션 3 (가사 생성) 출력
- "MV 만드는 방법" → 섹션 4 (비디오 생성) 출력
- "에러코드 목록" → 섹션 7 출력
- "어떤 모델 쓸 수 있어?" → 섹션 6 출력
- "폴링 어떻게 해?" → 섹션 8 출력
- "전체 API 목록" → 섹션 10 출력
- "크레딧 얼마야" → 섹션 1 출력

**Rules:**
- KIE_API_REFERENCE.md 파일 내용만 기반으로 답변
- API 키나 시크릿은 절대 출력하지 않음
- 문서에 없는 내용은 "KIE_API_REFERENCE.md에 해당 정보가 없습니다. https://docs.kie.ai/ 에서 확인하세요." 라고 안내
