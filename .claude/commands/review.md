코드 리뷰 및 보안 점검 스킬.

**Instructions:**
1. 변경된 파일 탐지:
   - `git diff HEAD` 또는 `git diff --cached`로 변경 사항 확인
   - 변경이 없으면 `$ARGUMENTS`로 지정된 파일을 리뷰

2. 코드 품질 검사:
   - 문법 오류 및 오타
   - 중복 코드
   - 하드코딩된 값 (매직 넘버, 문자열)
   - 에러 핸들링 누락
   - console.log 디버깅 코드 잔존

3. 보안 취약점 스캔:
   - API 키/시크릿 하드코딩 여부
   - XSS 취약점 (innerHTML, eval 등)
   - SQL 인젝션 가능성
   - CORS 설정 문제
   - 인증/인가 우회 가능성
   - 민감 정보 노출 (에러 메시지, 스택 트레이스)
   - Mixed Content: 외부 URL(아바타, 이미지 등)이 http://로 시작하면 _ensureHttps() 적용 필수
   - 인증 불필요한 클라이언트 코드에서 관리자 전용 API(401 반환) 호출 금지 → 공개 엔드포인트 사용

4. 성능 이슈:
   - 불필요한 DOM 조작
   - 메모리 누수 가능성 (이벤트 리스너 미정리)
   - N+1 쿼리 패턴
   - 대용량 데이터 미페이지네이션

5. 결과 보고:
   ```
   📝 코드 리뷰 결과

   🔴 Critical (즉시 수정 필요):
   - [파일:라인] 설명

   🟡 Warning (개선 권장):
   - [파일:라인] 설명

   🟢 Info (참고):
   - [파일:라인] 설명

   📊 요약: Critical N개, Warning N개, Info N개
   ```

**Rules:**
- 리뷰는 읽기만 수행, 코드 수정은 하지 않음
- 수정이 필요하면 구체적인 개선 코드를 제안
- false positive를 줄이기 위해 컨텍스트를 충분히 파악
