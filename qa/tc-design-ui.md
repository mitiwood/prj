# TC — 디자인 / UI

> 우선순위: P1 = 필수 / P2 = 중요 / P3 = 선택  
> 업데이트: 2026-04-01

---

## 테마

### TC-D01 · P1 · 다크/라이트 모드 전환
**스텝**:
1. 설정 → 테마 토글
2. 모든 주요 화면 순회 (생성, 히스토리, 커뮤니티)  

**기대결과**: 배경/텍스트/카드/버튼 색상 모두 올바르게 전환, 다크 배경이 라이트 모드에 남지 않음

---

### TC-D02 · P2 · 테마 유지 (새로고침)
**스텝**: 라이트 모드 설정 → 페이지 새로고침  

**기대결과**: 새로고침 후에도 라이트 모드 유지 (localStorage 보존)

---

### TC-D03 · P2 · 시스템 다크모드 감지
**스텝**: OS 다크모드 켜진 상태에서 첫 접속  

**기대결과**: 자동으로 다크 테마 적용

---

## 레이아웃 / 반응형

### TC-D04 · P1 · 모바일 폭 (375px ~ 480px)
**스텝**: DevTools에서 375px, 414px, 480px로 각각 확인  

**기대결과**: 스크롤바 없음, 텍스트 잘림 없음, 버튼 누를 수 있는 충분한 크기(최소 44px)

---

### TC-D05 · P2 · 데스크탑 폭 (768px↑)
**스텝**: 768px, 1024px에서 확인  

**기대결과**: max-width 480px 컨테이너 중앙 정렬, 양 옆 여백 처리

---

### TC-D06 · P1 · 결과 카드 가로 레이아웃
**스텝**: 트랙 생성 후 결과 카드 확인  

**기대결과**: 썸네일(88px) 좌측, 내용 우측 배치, 이미지 없을 때 🎵 플레이스홀더 표시, 카드가 화면 폭을 벗어나지 않음

---

### TC-D07 · P2 · 커버 이미지 없을 때
**스텝**: 이미지 URL 없는 트랙 생성 또는 이미지 로드 실패 시뮬레이션  

**기대결과**: 보라색 그라디언트 배경 + 🎵 아이콘 플레이스홀더, 레이아웃 깨짐 없음

---

## 오디오 플레이어

### TC-D08 · P1 · 재생 버튼 동작
**스텝**: 결과 카드 ▶ 버튼 클릭  

**기대결과**: 버튼 ▶→⏸ 전환, 진행 바 움직임, 타임코드 업데이트

---

### TC-D09 · P1 · 진행 바 클릭 탐색
**스텝**: 진행 바 중간 클릭  

**기대결과**: 해당 위치로 재생 위치 이동

---

### TC-D10 · P2 · 비주얼라이저
**스텝**: 재생 중 비주얼라이저 영역 확인  

**기대결과**: 28개 막대가 음악에 맞춰 애니메이션, 일시정지 시 정지

---

### TC-D11 · P1 · 미니 플레이어 표시
**스텝**: 히스토리/커뮤니티에서 트랙 재생  

**기대결과**: 화면 하단 미니 플레이어 고정 표시, 다른 탭 이동 시에도 재생 유지

---

### TC-D12 · P1 · 풀 플레이어 열기/닫기
**스텝**: 미니 플레이어 클릭 → 풀 플레이어 열림 → ✕ 닫기  

**기대결과**: 슬라이드업 애니메이션, 닫을 때 미니 플레이어 유지, 재생 유지

---

## 모달 / 바텀 시트

### TC-D13 · P1 · 모달 오버레이 닫기
**스텝**: 모달 열기 → 배경 클릭  

**기대결과**: 모달 닫힘, 배경 스크롤 잠금 해제

---

### TC-D14 · P1 · 바텀시트 스와이프 다운 닫기
**스텝**: 바텀시트 위에서 아래로 드래그  

**기대결과**: 시트 아래로 닫힘

---

### TC-D15 · P2 · 바텀시트 Z-index 충돌
**스텝**: 미니 플레이어 활성 상태에서 바텀시트 열기  

**기대결과**: 바텀시트가 미니 플레이어 위에 표시됨

---

### TC-D16 · P2 · 연장 모달 슬라이더
**스텝**: 연장 모달 → 슬라이더 드래그  

**기대결과**: 실시간 위치값 표시, 최솟값/최댓값 경계 동작 정확

---

## 로딩 / 에러 상태

### TC-D17 · P1 · 생성 중 로딩 카드
**스텝**: 생성 시작 직후 결과 영역 확인  

**기대결과**: 모드별 컬러 로딩 카드(커스텀=보라, 심플=초록, YouTube=파랑, MV=주황) + 스피너, 완료 시 실제 카드로 교체

---

### TC-D18 · P2 · 이미지 로드 실패
**스텝**: 네트워크에서 이미지 URL 차단 후 카드 확인  

**기대결과**: 이미지 대신 플레이스홀더, 카드 구조 유지

---

### TC-D19 · P2 · 토스트 메시지
**스텝**: 다운로드, 공유, 에러 등 토스트 발생 동작  

**기대결과**: 하단/상단 토스트, 2-3초 후 자동 사라짐, 중복 토스트 겹치지 않음

---

## 탭 / 네비게이션

### TC-D20 · P1 · 탭 전환 (생성/히스토리/커뮤니티/설정)
**스텝**: 각 탭 클릭  

**기대결과**: 활성 탭 강조, 화면 즉시 전환, 스크롤 위치 탭별 독립 유지

---

### TC-D21 · P1 · 결과 카드 탭 (재생/가사)
**스텝**: 결과 카드 "📝 가사" 탭 클릭  

**기대결과**: 가사 탭 내용 표시, "🎵 재생" 탭 비활성, 탭 하단 보라 밑줄

---

## 배치 헤더

### TC-D22 · P2 · 배치 접기/펼치기
**스텝**: 배치 헤더 ▼ 버튼 클릭  

**기대결과**: 0.3s ease 애니메이션으로 카드 영역 접힘, 버튼 180° 회전, 재클릭 시 펼침

---

### TC-D23 · P2 · 배치 삭제
**스텝**: 배치 헤더 ✕ 버튼 클릭  

**기대결과**: 해당 배치 전체(헤더 + 카드) DOM에서 제거, 다른 배치 영향 없음

---

## 가사 기능

### TC-D24 · P2 · 가사 편집 토글
**스텝**: 가사 탭 → ✏️ 가사 편집 클릭  

**기대결과**: 텍스트에어리어 표시, 기존 가사 로드, 저장/취소 버튼 노출

---

### TC-D25 · P1 · 가사 노래방 모드
**스텝**: 커버 이미지 클릭 → 노래방 모드  

**기대결과**: 전체화면 노래방 뷰 오픈, 가사 싱크 하이라이트, ESC/닫기 가능

---

### TC-D26 · P2 · 가사 복사
**스텝**: 가사 탭 → 📋 가사 복사  

**기대결과**: 클립보드에 가사 복사 성공 토스트

---

## 접근성

### TC-D27 · P3 · 버튼 최소 터치 영역
**스텝**: DevTools에서 모든 버튼 박스 크기 확인  

**기대결과**: 모든 인터랙티브 요소 44×44px 이상

---

### TC-D28 · P3 · 색상 대비
**스텝**: 라이트모드에서 텍스트/배경 대비 확인 (Chrome Accessibility)  

**기대결과**: WCAG AA 기준 (4.5:1) 이상
