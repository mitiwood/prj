## 2. 음악 생성

### 2.1 생성 플로우

```
1. 로그인 확인     → 미로그인 시 로그인 시트 표시
2. 게스트 한도     → 2곡 초과 시 차단
3. checkPlanLimit()→ 월 한도 초과 여부 (클라이언트)
4. serverCreditCheck() → 서버 크레딧 검증
5. API 호출        → /api/kie (또는 /api/kie-proxy)
6. 폴링            → 생성 완료까지 상태 체크
7. 완료            → trackUsage() + 크레딧 차감
8. 실패            → 1회 자동 재시도 (민감단어/크레딧/422 제외)
```

### 2.2 모드별 파라미터

#### Custom 모드

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| title | string | (없음) | 곡 제목 (선택) |
| prompt | string | (없음) | 가사/프롬프트 |
| genre | string[] | [] | 장르 선택 |
| mood | string[] | [] | 분위기 (최대 3개) |
| instrumental | boolean | false | 인스트루멘탈 여부 |
| vocal_style | string | "auto" | 보컬 스타일 |
| bpm | number | 120 | BPM (60~200) |
| duration | string | "~1min" | 길이 (~1분 / ~2분) |

**보컬 스타일 8종:** 자동 / 남성 / 여성 / 혼성 / 보이그룹 / 걸그룹 / 합창 / 래퍼

#### Simple 모드

| 파라미터 | 타입 | 설명 |
|---------|------|------|

> 400자 초과 시 Custom 모드로 자동 전환 (kie.ai 500자 제한 대응)

#### YouTube 모드

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| url | string | YouTube URL (필수) |

#### MV 모드

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| track_id | string | 기반 음악 (선택) |

| style | string | 영상 스타일 |

```

관리자: 필터 없이 전체 표시
```

### 4.2 상호작용

| 기능 | 로그인 | 게스트 | 설명 |
|------|--------|--------|------|
| 좋아요 | ✓ | ✗ | 실시간 전체 UI 동기화 |
| 싫어요 | ✓ | ✗ | 실시간 동기화 |
| 댓글 | ✓ | ✗ | 작성/삭제 |
| 팔로우 | ✓ | ✗ | sessionStorage 5분 TTL |
| 공유 | ✓ | ✓ | 링크/QR |
| 신고 | ✓ | ✗ | 트랙/댓글 신고 |

### 4.3 댓글 정책

```
- 삭제 권한: 작성자 본인 또는 관리자
```

```
- heartbeat 주기: 30초
- 온라인 기준: 최근 5분 이내 last_login
- 표시 위치: 커뮤니티 헤더
```

| 데이터 | TTL | 저장소 |

## 6. 결제 및 플랜

### 6.1 플랜 정의 (Single Source of Truth: `/api/toss-config.js`)

| 플랜 | 가격 | 기간 | 주요 혜택 |
|------|------|------|----------|
| **Free** | 무료 | 월 | 3곡, V3.5, ~1분 |
| **Pro** | 9,900원 | 월 | 50곡, MV 3개, ~3분, 보컬 변환 |
| **Creator** | 19,900원 | 월 | 무제한, 상업 라이선스, ~8분 |
| **Supervisor** | 무료 | 무기한 | 전 기능 무제한 + 관리자 권한 |

### 6.2 크레딧 팩 (일회성)

| 팩 | 가격 | 크레딧 |
|----|------|--------|
| 소형 | 1,900원 | 10 크레딧 |
| 중형 | 4,900원 | 30 크레딧 |
| 대형 | 12,900원 | 100 크레딧 |

### 6.3 결제 플로우 (Toss Payments)

```
1. 로그인 필수 확인
2. Toss 위젯 렌더링
3. 카드 결제 → Toss API 검증
4. Supabase users.plan 업데이트
5. localStorage 동기화
6. 월 사용량 초기화

환불/취소:
- Toss 웹훅: CANCELED/EXPIRED → plan='free' 자동 다운그레이드
- plan_expires 만료 시 서버 체크 → 자동 다운그레이드

플랜 동기화:
- 설정 탭 진입 시 서버와 즉시 동기화
- 클라이언트/서버 2중 검증
```

### 6.4 플랜 검증 체계

```
클라이언트: checkPlanLimit() — 즉각 UI 피드백
서버: /api/check-credit — 실제 차감 전 최종 검증
적용 대상: generate(), generateMV(), YouTube 모드, lyrics, vocal, extend
```

## 10. 보안

### 10.1 인증 보안

```
- OAuth 2.0 (Google/Kakao/Naver)
- 서버: ADMIN_SECRET Bearer 토큰
- RLS: Supabase Row Level Security
  - anon: 읽기만 허용
  - service_role: 쓰기 허용
```

### 10.2 XSS 방지

```
- 모든 사용자 입력: esc() 함수로 HTML 이스케이프
- 아바타 URL: _ensureHttps()로 http→https 변환
```

### 10.3 API 보안

```
- KIE_API_KEY: 환경변수 관리, 프론트 직접 노출 금지
- 일반 사용자: 관리자 전용 API (/api/users GET) 직접 호출 금지
  → ?action=public-names 경유 사용
- Mixed Content: 외부 이미지 반드시 https 강제
```
