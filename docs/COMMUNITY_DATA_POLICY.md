# 커뮤니티 데이터 정책 (확정)

> 2026-03-24 확정. 이 정책을 변경하려면 반드시 사용자에게 먼저 확인할 것.

---

## 핵심 규칙

**커뮤니티의 모든 콘텐츠는 실제 가입 사용자(users 테이블)의 트랙만 표시한다.**

- `users` 테이블에 없는 owner의 트랙은 절대 불러오지 않음
- 더미/테스트/봇 트랙은 커뮤니티에 표시하지 않음

## 적용 범위

| 영역 | 필터 적용 |
|---|---|
| 주간 TOP 10 차트 | O |
| 리더보드 | O |
| 크리에이터 카드 리스트 | O |
| 스포트라이트 (추천 크리에이터) | O |
| 히어로 (인기곡) | O |
| 팔로잉 피드 | O |
| 커뮤니티 전체 트랙 | O |

## 구현 방식

```
1. _loadRealUsers() — /api/users에서 실제 사용자 이름 Set 캐시
2. _isRealUser(name) — 캐시에 있는지 확인
3. _sbTracks 로드 시 .filter(t => _isRealUser(t._owner?.name)) 적용
4. 폴링 갱신 시에도 동일 필터 적용
```

## 주의사항

- **이 정책을 수정하려면 반드시 사용자에게 먼저 확인**
- 새 사용자가 가입하면 자동으로 users 테이블에 추가 → 커뮤니티에 표시됨
- 더미 트랙은 DB(tracks 테이블)에는 남아있지만 프론트에서 필터링됨
- 관리자 페이지(admin)에서는 필터 없이 전체 트랙 표시
