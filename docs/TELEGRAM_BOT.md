# 텔레그램 봇 명령어 가이드

> 슬래시(/) 없이 바로 입력 가능. 자연어도 지원.

---

## 🎵 음악 생성

```
생성 <가사 또는 설명>
생성 -t 제목 -s 스타일 -v m 가사내용
```

**옵션:**
- `-t <제목>` 곡 제목 (기본: AI 자동)
- `-s <스타일>` 장르/분위기 (예: k-pop, ballad, lo-fi)
- `-v <m|f>` 보컬 성별 (m=남, f=여)
- `-i` 인스트루멘탈 (보컬 없음)
- `-m <모델>` V4_5(기본), V5

**예시:**
```
생성 봄날의 따뜻한 고백 노래
생성 -t 봄바람 -s k-pop, ballad -v f 벚꽃이 흩날리는 봄날에
생성 -i -s lo-fi, chill 비오는 날 카페
심플 비 오는 날 듣기 좋은 재즈
커스텀 [Verse 1] 너를 만난 그날부터
```

**동작 흐름:**
1. 봇이 kie.ai API에 생성 요청
2. 최대 45초간 폴링 (Vercel 타임아웃 안전 마진)
3. 45초 내 완료 → 즉시 결과 전송 + Supabase 저장 + 커뮤니티 공개
4. 45초 초과 → "생성 중" 알림 + kie.ai 콜백(/api/callback)으로 자동 완료 처리

## 트러블슈팅

### 봇이 응답하지 않을 때

1. **웹훅 상태 확인:**
   ```
   GET /api/tg-webhook?action=info
   Authorization: Bearer {ADMIN_SECRET}
   ```

2. **`last_error_message: "Read timeout expired"`인 경우:**
   - 원인: 장시간 작업(음악 생성 등)이 Vercel 60초 타임아웃 초과
   - 해결: 웹훅 재설정
   ```
   GET /api/tg-webhook?action=remove  (밀린 메시지 클리어)
   GET /api/tg-webhook?action=set     (재등록)
   ```

3. **`pending_update_count`가 높은 경우:**
   - 밀린 메시지가 쌓여 있음
   - `action=remove`로 웹훅 해제하면 자동 클리어됨
   - 이후 `action=set`으로 재등록

### 아키텍처 (타임아웃 방지)

```
텔레그램 → POST /api/tg-webhook
                ↓
         즉시 200 응답 (텔레그램에 OK 반환)
                ↓
         백그라운드로 명령 실행
                ↓
         짧은 명령: 즉시 결과 전송
         긴 명령(/생성): 45초 폴링 → 미완료 시 콜백 대기
                ↓
         kie.ai 콜백 → POST /api/callback
                ↓
         트랙 저장 + 텔레그램/카카오 알림
```

### 환경변수

```
필수: TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, KIE_API_KEY
      SUPABASE_URL, SUPABASE_SERVICE_KEY
선택: ADMIN_SECRET, GITHUB_TOKEN, ANTHROPIC_API_KEY
```
