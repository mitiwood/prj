## 1. 트랙 저장/동기화 — 크리티컬 버그 수정

### co_owner_name 컬럼 부재 → GET 전체 실패
- `api/tracks.js` GET에서 `co_owner_name` 컬럼 참조 → Supabase 400 에러
- **모든 사용자의 트랙 조회가 메모리 폴백**으로 떨어짐 → 기존 트랙 안 보임
- owner 필터에서 미존재 컬럼 참조 제거로 해결

### parent_id/co_owner 컬럼 부재 → POST 전체 실패
- `api/tracks.js` POST에서 `parent_id`, `collab_id`, `co_owner_*` 참조
- **모든 트랙 저장이 Supabase에 안 되고 메모리에만 저장** → 서버 재시작 시 소실
- 미존재 컬럼 제거로 해결

### 로컬→서버 트랙 동기화
- `_loadMyTracks`에서 로컬에만 있는 트랙을 서버에 자동 동기화 (`_sync: true`)
- `_saveTrackToServer` 실패 시 3초 후 1회 자동 재시도
- 동기화 시 `notifications` 테이블에 시스템 로그 기록
- 저장 실패 시에도 실패 로그 기록 (MZ톤 메시지)

### 히스토리 뷰 갱신 누락
- `_loadMyTracks` 완료 후 `renderHistoryView()` 호출 누락 수정

## 2. 커뮤니티 성능 최적화

### 데이터 로딩 지연
- 페이지 로드 시 커뮤니티 프리로드 **제거** → 탭 진입 시로 지연
- localStorage 캐시(`kms_comm_cache`, `kms_myfeed_creators`) **완전 제거**
- 메모리 캐시만 사용, 캐시 TTL 30초 → **5분**
- 폴링 간격 30초 → **60초**
- 메모리에 데이터 있으면 **0ms 즉시 렌더** + 백그라운드 갱신
- `_sbLoading` 동시 호출 시 Promise 대기 (빈 화면 방지)

### 캐시 기반 렌더링
- fingerprint 변경 감지 → 변경 없으면 리렌더 스킵
- 탭 전환 시 `force=true` → 캐시 우회하고 해당 모드 데이터 새로 렌더

### 로딩 인디케이터 통일
- `_commLoadingHtml(msg, small)` 헬퍼 함수
- 보라색 바운스 도트 3개 + 텍스트 (full/small 두 사이즈)
- 커뮤니티, 팔로잉, 채팅, 크리에이터, MY피드, 댓글 모두 통일
- 최초 진입 시 텍스트 없이 도트만 표시

### 순차 페이드인 애니메이션
- `commFadeUp`: translateY(20px)+scale(.97) → cubic-bezier(.16,1,.3,1) 스프링
- 트랙/차트/크리에이터 아이템 30~40ms 간격 stagger
- 히어로 fadeIn, 탭 전환 시 부드러운 전환

## 6. 커뮤니티 채팅

### 채팅 모드
- 진입 시 채팅 UI + 크리에이터만 표시
- DJ, 차트, 무드, 스포트라이트, 장르, 챌린지, 추천, 활동피드 **DOM 완전 제거**

- 무한 스크롤 IntersectionObserver 해제

### 메시지 가로 배치
- 이름(inline) + 메시지 내용 + 시간(inline)을 버블 안 한 줄에 표시

### 타이핑 인디케이터
- 입력 시 서버에 typing 알림 (4초 TTL)
- 다른 사용자 "○○님이 입력 중..." 표시

### 중복 메시지 방지
- 폴링 시 `data-msg-id`로 DOM에 이미 있는 메시지 스킵

### Supabase Realtime 시도
- WebSocket 기반 실시간 전환 시도
- anon key 줄바꿈 문제 수정 (서버+클라이언트)
- 최종: 폴링 1초 + DB 기반 타이핑으로 안정화

### 봇 명령어
- `채팅공지 <메시지>` — 관리자 공지 전송 (📢 prefix)
- `채팅초기화` / `clearchat` — chat_messages 전체 삭제
